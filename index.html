개념설명게임에서 보기
상태 (State)에이전트가 관찰하는 환경 정보차에서 뻗는 7개 레이더 센서 + 현재 속도 = 8차원 벡터
행동 (Action)에이전트의 선택지조향 5단계 (급좌회전 ~ 급우회전)
보상 (Reward)행동의 좋고 나쁨 신호전진 +, 충돌 −. 왼쪽 슬라이더로 직접 설계
DQNQ함수를 신경망으로 근사하는 심층 강화학습8→16→16→5 신경망, 외부 라이브러리 없이 순수 JS 구현
ε-greedy 탐험확률 ε로 무작위 행동 → 새로운 전략 발견상단 ε 게이지. 초반엔 마구 돌다가 점점 배운 대로 주행
경험 재현
(Replay Buffer)		과거 경험을 저장했다가 무작위로 꺼내 학습 → 상관관계 제거상단 "경험 버퍼" 카운터 (5만 개 저장)
타깃 네트워크학습 목표를 천천히 갱신해 발산 방지1,500스텝마다 동기화 (학습 일지에 기록)
Double DQN행동 선택과 평가를 분리해 Q값 과대평가 방지학습 업데이트에 적용됨 (van Hasselt, 2015)
Huber 손실큰 오차의 그래디언트를 제한해 안정화TD 오차 클리핑으로 구현
할인율 γ미래 보상의 현재 가치슬라이더로 조절 — 낮추면 근시안적 주행
보상 해킹잘못 설계된 보상의 허점을 파고드는 현상전진 보상 0 + 속도 보너스만 주면? 직접 실험!
일반화학습한 정책이 새로운 환경에서도 통하는가트랙 편집 후 "모델 유지" — 처음 보는 트랙 주행
커리큘럼 러닝쉬운 과제부터 차례로 학습오벌 → 클래식 → 서펜트 순서로 훈련해 보세요
절단 vs 종료시간 초과(truncation)는 실패(termination)와 다름시간 초과 시 done=0으로 저장해 부트스트랩 유지
PERTD 오차가 큰 "중요한 실수"를 더 자주 복습 (Schaul, 2016). sum-tree로 O(log n) 샘플링하이퍼파라미터의 PER 토글 + ⚗️ 벤치마크
n-step 리턴여러 스텝의 보상을 묶어 학습 신호 전파를 가속1/3/5-step 선택
희소 보상보상이 드물면 탐험이 극도로 어려워짐 — 현실 RL의 최대 난제 중 하나보상 설계의 🌵 모드
